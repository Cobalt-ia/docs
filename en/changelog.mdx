Cobalt changelog and product updates. Stay up to date with new features, improvements, and bug fixes shipped in Cobalt.

<Update label="June 2, 2026" description="Meeting workflow, update block, project email, Chrome extension">
  ### Default meeting workflow

  An **internal workflow** is now attached to every organization: after a meeting, it **generates the transcript**, **enriches** records, and **creates follow-up tasks** automatically.

  <img src="/images/changelog-workflow-meeting.png" alt="Internal meeting workflow: transcription, enrichment, follow-up tasks" width="760" />

  ### New workflow block: update

  A new block lets you, after other nodes, **update any field** of a record — candidate, contact, opportunity, company…

  <img src="/images/bloc-modification-candidat.png" alt="Update block: update the candidate" width="420" />

  <img src="/images/bloc-modification-entreprise.png" alt="Update block: update the company" width="420" />

  ### Send an email from a project (no sequence)

  You can now **send an email to contacts in a project** (clients or candidates) **without attaching a sequence**.

  <img src="/images/changelog-envoi-mail-projet.png" alt="Send an email to selected contacts in a project" width="520" />

  ### New Chrome extension version

  - Fixed the bottom-of-page extension issues for some users.
  - **Remaining credits** now visible directly in the extension.

  <img src="/images/changelog-extension-v2.png" alt="New version of the Cobalt Chrome extension" width="360" />
</Update>

<Update label="June 1, 2026" description="New features and fixes">
  ### Project status and filters

  Projects now have an **Active** or **Archived** status. Archive finalized projects: they no longer appear by default in the list, keeping the projects page focused on what's ongoing. You can also **filter projects by type and by member** — useful in weekly meetings to review everyone's business actions for the week.

  ### MCP documentation in Cobalt

  Cobalt's documentation is now available via **MCP** directly in the app: users can **ask their questions directly to Cobalt**, like a FAQ.

  ### Add clients and companies from a project

  You can now **add new clients (contacts) and new companies directly from a project**, without having to create them first in the associated table.

  <img src="/images/changelog-nouveau-contact.png" alt="Add a new contact from a project" width="520" />

  <img src="/images/changelog-nouvelle-entreprise.png" alt="Add a new company from a project" width="520" />

  ### New workflow block: create a manual task

  A new workflow block lets you **create a manual task** automatically — handy to generate a reminder, for example "call a client".

  <img src="/images/bloc-action-manuelle.png" alt="Workflow block: create a task" width="420" />

  ### Fixes

  - Fixed the **AI chat**.
  - Fixed **meeting transcription**.
  - Fixed **signature creation**.
</Update>

<Update label="May 29, 2026" description="Filters + tasks and transcriptions visibility">
  ### Improved filter management

  You can now **remove an applied filter directly** from the filter bar by clicking the cross on the filter — no need to reopen the menu. A **Reset** button also clears all filters at once.

  <img src="/images/changelog-filtres.png" alt="Filter bar with direct filter removal and a Reset button" />

  ### Tasks and transcriptions visibility

  A **new organization permission** controls the visibility of **tasks** and **transcriptions** (meetings and phone calls): each member can now see only **their own** tasks and transcriptions. Previously, these were visible to everyone by default.
</Update>

<Update label="May 28, 2026" description="New Excel view on projects">
  ### "Excel" view on projects

  Projects now have a new **table ("Excel") view**: your contacts show row by row with their columns (name, title, company, LinkedIn, email, phone, status). This view is now the **default for prospecting projects**.

  <img src="/images/changelog-vue-excel.png" alt="Excel view of a prospecting project" />

  Three views are now available on projects:

  <img src="/images/changelog-3-vues.png" alt="Project view switcher with three views" />
</Update>

<Update label="May 27, 2026" description="LinkedIn Chrome Extension + quality improvements">
  ### LinkedIn Chrome Extension available

  Cobalt's **LinkedIn Chrome Extension** is now available. From a LinkedIn profile, it **enriches the email (work and personal) and phone number**, then lets you add them straight to your **CRM/ATS**, to an **existing project**, or **create a new project** on the fly.

  To install it, go to your **personal settings** and click **Install the Cobalt LinkedIn extension to get started**: the extension installs and syncs with your app.

  <img src="/images/extension-chrome.png" alt="Install the Cobalt LinkedIn extension from settings" />

  ### Quality improvements

  - Improved quality of **enrichment** and **tasks** created from a **meeting** or **phone call** transcription.
  - Improved **People Search Engine** quality.
</Update>

<Update label="May 26, 2026" description="Resend the link to a candidate">
  ### Resend the link to a candidate

  You can now **resend the link** to a candidate even if it was already sent. Handy when the candidate can't find the original email.

  <img src="/images/changelog-renvoyer-lien.png" alt="Invitation sent dialog with the Resend email button" width="440" />
</Update>

<Update label="May 25, 2026" description="Automatic skills document creation">
  ### Skills document created when a candidate is created

  By default for all organizations: if you have a **default template**, the **skills document** is now created automatically as soon as a candidate is created — saving time on every add.

  <img src="/images/changelog-creation-dossier.png" alt="Setting to automatically create the skills document from the default template" width="620" />
</Update>
