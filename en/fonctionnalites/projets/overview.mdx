**Projects** structure your work in Cobalt. Each project groups the records and actions tied to a specific goal — a need to fill, a consultant to reposition, an account to grow, and so on.

## Create a project

You can create a project in two ways:

- From the **Projects page**, using the **New project** button.
- From the **People Search Engine**, when you want to **save people** you've sourced or prospected into a project.

When creating it, enter a **name** and a **use case**:

<img src="/images/projet-creation-modal.png" alt="New project modal: project name and use case" width="560" />

Depending on the **use case** you pick, Cobalt will ask for **different information** and adapt the project's default view:

## Views

A project can be displayed in three views:

- **Cards** — card layout.
- **Kanban** — columns by status. This is the **default for sourcing projects**.
- **Excel** — row-by-row table. This is the **default for prospecting projects**.

## Project status

Each project has an **Active** or **Archived** status. Active projects are the ongoing ones — potentially worth following. Once a project is finalized, **archive it**: it no longer appears by default in the projects list, keeping the page free of projects you no longer use.

## Filters

You can **filter projects by type and by member**. Handy, for example, in a weekly meeting: filter by member to review everyone's business actions for the past week.

## Send a message

From a project's **list of people**, you can **send a message**:

- **Manually**, straight from the project.
- **Automatically** via [workflows](/en/fonctionnalites/creer-premier-workflow) *(coming soon)*.

For now, sending is **email only** — multichannel sending is not available yet.

## Attachments

Depending on the project type, you can **attach needs or candidates**.

You can also **attach workflows** to partially or fully automate certain project tasks. See [Workflows overview](/en/fonctionnalites/creer-premier-workflow) to learn more.

## Permissions

By default, **everyone can see and edit projects**. If you want, you can restrict this access using custom roles — see [Roles and permissions](/en/demarrer/configurer-espace).
