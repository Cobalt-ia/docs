A Cobalt workflow is built by assembling blocks. This page lists the available blocks.

## <Icon icon="zap" /> Trigger block

The trigger block defines the event that starts the workflow. Every workflow begins with a trigger.

### <Icon icon="play" /> Manual trigger

The workflow is started manually by a user.

### <Icon icon="clock" /> Scheduled trigger

The workflow runs on a schedule (time, day, recurrence).

### <Icon icon="refresh-cw" /> Update trigger

The workflow runs when a field or record is updated.

### <Icon icon="sparkles" /> Creation trigger

The workflow runs when a record is created (candidate, need, company, etc.).

## <Icon icon="hourglass" /> Wait block

Pauses the workflow for a defined duration before moving to the next step.

## <Icon icon="search" /> Search block

Searches for profiles, for sourcing or prospecting.

## <Icon icon="globe" /> Internet search block

Searches any useful information on the internet that can be used later in the workflow.

For example: find job postings similar to a candidate you want to place, identify the authors of those postings, find them on LinkedIn, and surface them as **prospects** who might be interested in meeting that candidate.

## <Icon icon="pencil" /> Update block

Updates any field of a record: candidate, client, need, company, etc.

## <Icon icon="send" /> Communication block

Sends a message through the channel of your choice:

- Send an email
- Send a LinkedIn connection request
- Send a LinkedIn message
- Send a WhatsApp message
- …

## <Icon icon="list-todo" /> Manual action block

Creates a manual task for a user — for example, calling someone.

## <Icon icon="workflow" /> Call-a-workflow block

Calls another workflow from the current one.
