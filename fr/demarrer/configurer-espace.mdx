Avant que votre équipe commence à sourcer et à envoyer des séquences d'outreach, prenez quelques minutes pour configurer votre espace Cobalt. Une configuration complète garantit que chaque dossier de compétences porte vos couleurs, que chaque séquence part de la bonne adresse email et que vos données candidats existantes sont disponibles dès le premier jour.

## Profil entreprise

Votre profil entreprise est le socle de votre espace Cobalt. Pour le mettre à jour, allez dans **Paramètres → Espace de travail**.

- **Nom de l'entreprise** — apparaît dans les dossiers de compétences, les pieds de page des emails et les enregistrements internes.
- **Logo** — importez un fichier PNG ou SVG. Cobalt utilise votre logo automatiquement dans tous les dossiers de compétences et modèles générés.
- **Langue par défaut** — définit la langue pour le contenu généré automatiquement, les modèles d'emails et les réponses de l'agent IA Balt.

Les modifications apportées à votre profil entreprise s'appliquent immédiatement à l'ensemble de l'espace de travail.

## Rôles et permissions

Gérez les rôles et les permissions des membres de votre organisation depuis **Paramètres → Rôles & permissions**.

Trois rôles système sont fournis par défaut :

- **Propriétaire** — toutes les permissions.
- **Administrateur** — accès étendu à la gestion de l'espace.
- **Membre** — accès standard aux fonctionnalités du quotidien.

### Créer un rôle personnalisé

Si besoin, créez un **rôle personnalisé** (bouton **Nouveau rôle**) pour définir précisément les permissions sur l'ensemble des possibilités de l'application.

**Périmètre d'accès**

- **Tous** — l'utilisateur peut accéder et agir sur tous les enregistrements, y compris ceux créés par d'autres membres.
- **Propres** — l'utilisateur peut uniquement accéder et agir sur les enregistrements qu'il a lui-même créés.

**Permissions par entité** — pour chaque entité, vous définissez les droits **Voir**, **Créer**, **Modifier** et **Supprimer** : Candidats, Entreprises, Contacts, Opportunités, Positionnements, Réunions, Projets et Tâches.

**Permissions de l'organisation** — activez ou non chaque droit :

- Inviter des membres
- Retirer des membres
- Modifier les rôles des membres
- Modifier l'organisation
- Gérer les intégrations
- Gérer les ressources
- Accéder aux informations légales des candidats

## Modèles de dossiers de compétences

Cobalt génère des dossiers de compétences (dossiers de compétences) personnalisés à vos couleurs. Pour que chaque document soit conforme à votre charte dès le premier export, configurez vos modèles avant de commencer à en générer.

Allez dans **Paramètres → Modèles de documents** pour :

- **Importer votre logo** — si vous ne l'avez pas encore fait dans le profil entreprise, ajoutez-le ici.
- **Définir les couleurs et polices** — choisissez les couleurs primaires et secondaires de votre marque ; sélectionnez parmi les familles de polices disponibles.
- **Définir un modèle par défaut** — le modèle par défaut est utilisé chaque fois que vous générez un document sans en sélectionner un spécifique.

Vous pouvez créer plusieurs modèles sur les plans payants, ce qui vous permet de personnaliser les documents par client, par entité ou par type de poste. Par exemple, vous pouvez maintenir un modèle pour les profils IT et un autre pour les placements cadres.

<Tip>
  Créez un modèle spécifique pour vos comptes clients les plus importants. Un document qui correspond à l'identité visuelle du client se distingue et témoigne de votre attention aux détails.
</Tip>

## Connexion email

Connecter votre compte email permet à Cobalt d'envoyer des séquences d'outreach en votre nom et de rediriger les réponses vers votre boîte Cobalt. Cette connexion est **requise** pour utiliser les séquences multicanal.

Pour connecter votre email, allez dans **Paramètres personnels → Comptes email** :

- **Gmail (Google Workspace)** — cliquez sur **Connecter Gmail** et autorisez l'accès via le flux OAuth standard de Google. Cobalt ne demande que les permissions d'envoi et de lecture en votre nom. Voir [Google Workspace](/fr/integrations/google-workspace) pour les détails.
- **Outlook (Microsoft 365)** — cliquez sur **Connecter Outlook** et connectez-vous avec vos identifiants Microsoft. Voir [Microsoft 365](/fr/integrations/microsoft-365) pour les détails.

Une fois connecté, tous les emails de séquence partent de votre adresse personnelle — pas d'une adresse générique de la plateforme — ce qui maintient la délivrabilité et rend les échanges authentiques.

<Info>
  Chaque membre de l'équipe doit connecter son propre compte email individuellement. La connexion email d'un administrateur ne s'applique pas aux autres utilisateurs.
</Info>

## Synchronisation de l'agenda

Synchroniser votre agenda permet à l'agent IA Balt de rejoindre vos réunions automatiquement et de les transcrire en temps réel. Après chaque appel, Balt met à jour la fiche candidat ou client concernée dans votre CRM avec un résumé et les points d'action clés.

Pour connecter votre agenda, allez dans **Paramètres personnels → Comptes email** :

- **Google Calendar** — autorisez via le flux OAuth de Google. Balt détecte les liens Google Meet dans vos événements d'agenda et les rejoint automatiquement.
- **Microsoft Calendar** — autorisez via Microsoft. Balt rejoint les réunions Microsoft Teams de la même façon.

<Note>
  La synchronisation de l'agenda est disponible sur les plans payants. La transcription et la mise à jour automatique du CRM sont incluses sur tous les plans payants.
</Note>

## Import de données existantes

Cobalt est plus efficace quand il démarre avec vos données candidats et clients existantes. Vous pouvez importer depuis un fichier CSV ou migrer directement depuis un autre ATS — l'équipe Cobalt accompagne les migrations sans frais supplémentaires.

### Import CSV

Exportez vos données depuis votre outil actuel en fichier CSV, puis allez dans **Paramètres → Import → Importer un CSV**. Cobalt fait la correspondance des colonnes avec les bons champs et signale les lignes qui nécessitent votre attention avant de finaliser l'import.

### Migration ATS

Si vous migrez depuis l'une des plateformes suivantes, Cobalt gère la migration pour vous :

| ATS / CRM pris en charge |
|---|
| Bullhorn |
| Loxo |
| Vincere |
| Recruit CRM |
| RecruiterFlow |
| Ezekia |
| Greenhouse |
| Workable |
| Lever |
| SmartRecruiters |
| Manatal |
| JobAdder |
| Zoho Recruit |

Contactez le support Cobalt pour lancer une migration. L'équipe reste disponible tout au long du processus.

<Tip>
  Si votre organisation utilise BoondManager, connectez l'intégration **avant** d'importer des données. La synchronisation bidirectionnelle rapatriera automatiquement vos enregistrements existants, ce qui évite les doublons et vous épargne l'étape d'import. Voir [BoondManager](/fr/integrations/boondmanager).
</Tip>
