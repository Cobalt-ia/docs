Un workflow Cobalt se construit en assemblant des blocs. Cette page présente les différents blocs disponibles.

## <Icon icon="zap" /> Bloc de déclenchement

Le bloc de déclenchement définit l'événement qui démarre le workflow. Tout workflow commence par un déclencheur.

### <Icon icon="play" /> Déclencheur manuel

Le workflow est lancé manuellement par un utilisateur.

### <Icon icon="clock" /> Déclencheur planifié

Le workflow s'exécute selon une planification (heure, jour, récurrence).

### <Icon icon="refresh-cw" /> Déclencheur de modification

Le workflow se déclenche lorsqu'un champ ou un enregistrement est modifié.

### <Icon icon="sparkles" /> Déclencheur de création

Le workflow se déclenche lors de la création d'un enregistrement (candidat, besoin, entreprise, etc.).

## <Icon icon="hourglass" /> Bloc d'attente

Met le workflow en pause pendant une durée définie avant de passer à l'étape suivante.

## <Icon icon="search" /> Bloc de recherche

Recherche des profils, pour le sourcing ou la prospection.

## <Icon icon="pencil" /> Bloc de modification

Modifie n'importe quel champ d'un enregistrement : candidat, client, besoin, entreprise, etc.

## <Icon icon="send" /> Bloc de communication

Envoie un message via le canal de votre choix :

- Envoyer un email
- Envoyer une demande de connexion LinkedIn
- Envoyer un message LinkedIn
- Envoyer un message WhatsApp
- …

## <Icon icon="list-todo" /> Bloc d'action manuelle

Crée une tâche manuelle pour un utilisateur — par exemple, appeler quelqu'un.

## <Icon icon="workflow" /> Bloc d'appel d'un workflow

Appelle un autre workflow depuis le workflow courant.
